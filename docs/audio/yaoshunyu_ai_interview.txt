⚠ 常见误解 / 认知纠偏

「老登」≠ 年龄。姚顺宇反复强调：老登是一种人格——德不配位、仗着话语权否定年轻人、无法被实践检验却自封权威。靠谱的特质不会随年龄消失；好特质与生俱来，专业技能可以学，人格不能。「英雄主义已死」≠ 没有杰出个人，而是警惕把复杂集体成果简化为单一神话叙事。

接下来，第三部分，一、行业格局：从追能力到赌定义。

AI 下半场：担心的事变了

核心机制，一年前大家还担心「能不能追上 OpenAI」；现在 OpenAI、Anthropic、Google 三家都不担心追不上，更难的是想明白要做什么——这是一个 bet，需要人的 insight。

关键理解，模型在纸面上已高度同质化（SWE-bench 等差距多在 1-2 个百分点的噪声），但实际使用体验仍有分化：Claude 擅长 Agent/工具调用，Codex 在纯 coding 追赶，Gemini 在 reasoning 和日常场景表现好。

典型场景，选模型时不只看榜单，要按场景实测——写代码、做 Agent、日常对话的需求不同。

相关概念：上半场 = 能力竞赛；下半场 = 问题定义 + 数据 + 工程细节的意愿博弈

接下来，第四部分，二、技术纵深：Pre-train、Coding 与蒸馏。

Pre-train 没有到头

核心机制，Scaling law 仍有效，预训练远未饱和。瓶颈从「算力不够」转向「数据质量、问题定义、训练细节」——很多模型差异来自想象不到的工程细节（如 GitHub 数据天然质量更高）。

关键理解，「Pre-train 也是一种 RL」——用有限上下文训练，使用时接近无限上下文（train-with-finite-context, use-as-infinite-context）。

适用边界，不适用于已放弃继续 scale 的小团队；对头部实验室仍是主战场。

原文依据：「我觉得 Pre-train 没有到头……模型做到 train-with-finite-context, use-as-infinite-context。」

为什么 Coding 爆发最快

核心机制，① GitHub 提供天然高质量、可验证的数据；② 代码对错可自动检验（编译、测试），形成 RL 闭环；③ Agent 写代码让产品形态变简单——Google 内部约 90% 代码由模型产生。

典型场景，下一个爆发点可能是机器人（数据难、验证难）或更长 horizon 的 Agent 任务。

相关概念：除 coding 外，尚无真正 AI 原生的超级应用场景（ChatGPT 仍偏搜索延伸）

接下来，第五部分，三、组织与文化：Anthropic vs Google。

嘉宾观点Anthropic 的 Top-down 为何罕见

实行 top-down 需要技术一号位同时是公司决策人且有公信力——Dario/Jared/Sam 自己拍板。OpenAI 在 Ilya 离开后难做到；Google 是传统 bottom-up，研究员自己决定方向。

Startup 的优势是 make-a-bet：快速决策、强力推进。大公司优势是两面下注、任何方向成了都能跟上。

「闪超被（Anthropic）能够实行比较 top-down 的机制是一个很独特的事……技术的 leader 有公信力很重要，与此同时 CEO 没有成为一个阻力。」

离开 Anthropic 的动机与反思

核心机制，2025 年 9 月离开是为「学不一样的东西」，而非领导项目。当时对 Anthropic 偏悲观（API 卖 token 是差生意，只有 Google 能扛价格战），但后来 Claude Code 等产品巧思证明过度悲观。

关键理解，不后悔——个人动机是自我提升，选择没有错。OpenAI 被 Google 揍了一拳，Anthropic 自己上道了。

原文依据：「我离开的时候对这公司其实挺悲观的，但后来显然是我过度悲观了。」

相关概念：API 生意 vs 产品生意；差生意只有有完整产业链的公司能扛

大公司做不出 OpenClaw 的原因

核心机制，Google 等大厂无法上线需要用户让渡全部电脑权限的产品——法律风险、品牌风险、资源分配流程长。个人开源项目无此负担。

典型场景，2026 趋势可能是个人/小团队先展示可能性，大厂随后打磨跟进。

原文依据：「你 Google 是不可能提供这样的产品出去的……但是对于个人来说无所谓，我反正是一个开源的项目。」

接下来，第六部分，四、物理底色与思维迁移。

从理论物理到 AI 的跨界逻辑

核心机制，清华本科做凝聚态/非厄米系统，斯坦福博士做高能物理与量子信息。博士毕业后两周博后即离职转 AI——「总想挑战一些自己不太会的事」。

关键理解，物理训练的是在不同能量尺度下找有效理论（重整化群）——经典物理是量子物理在大尺度下的退化。这种「多尺度抽象」思维有助于理解 AI 系统的层次结构。

典型场景，跨学科不是劣势，深水区可能需要物理/数学高阶知识。

相关概念：姚期智也是物理转计算机——「从物理转到计算机后发现世界上没有什么难题了」

接下来，第七部分，五、技术预测与前沿方向。

6-12 个月：AI 端到端做研究

核心机制，当前 AI 已能加速实验（写代码、跑实验），但完整链条——写 code ，得到 跑实验 ，得到 看结果 ，得到 分析 ，得到 提出新假设 ，得到 设计新实验——尚未打通。未来半年到一年这条链将逐步完整。

适用边界，「AI 自己做实验」定义仍模糊；目前是对人类研究者的加速，非完全自主。

原文依据：「未来六到十二个月 AI 就会自己做实验……这条链条目前还没有完整，但我觉得这条链条可能是下一步会慢慢变得完整的事。」

世界模型：一万个人有一万个定义

核心机制，Google 的「世界模型」偏 end-to-end 视频/场景生成（给定状态+动作预测下一状态）；李飞飞等 lab 的定义可能完全不同。Continuous learning 与 long-context 管理无本质区别——KV cache 也是一种权重。

相关概念：范氏级变革 vs 有价值但非范式的进展（long horizon、硬件-模型协同）

接下来，第八部分，总结与行动。

AI 下半场核心从「能不能做到」变为「做什么、怎么定义」——模型能力已拉平，赌的是 insight 与工程细节

个人英雄主义时代结束：靠谱、细致、负责任比智商更重要；警惕神话个体的叙事

壳公司难逃脱模型引力：要么极速增长+自研模型，要么切入模型看不上的小市场

Pre-train 远未到头，Coding 因数据可验证而率先爆发；硬蒸馏是战略懒惰，聪明蒸馏是技术前沿

组织文化决定竞争力：Anthropic 的 top-down 需要技术一号位有公信力，Google 的 bottom-up 适合两面下注

选 AI 工具时按场景实测（coding / Agent / 日常），不只看公开 benchmark 排名

建立自洽的技术理解框架，用客观结果检验观点，而非迎合「老登」权威

若做 AI 产品，先想清楚：能否在模型公司反应前长大，或切入足够小的垂直场景

关注 long-context 与 Agent 端到端研究闭环的进展，这是未来 6-12 个月的关键变量

培养「靠谱」特质：做事细致、对交付负责，比追逐风口更重要

以前认为 AI 行业靠天才英雄和个人突破驱动；现在理解了这是一个集体主义、工程细节和问题定义的时代——每个人都是冲浪者，重要的是读懂浪的方向，用靠谱和自洽的观点在客观标准下赢得尊重，而非在社交媒体站队或神话个体。
